## 📌 Обзор

Перед нашей командой встала задача реализовать удаленное управление ровером, осуществление контроля всех его систем, выставление правил и протоколов их работы. 

Данный проект дает возможность полного контроля над композитным ровером для задач дальнего следования, обеспечивает оператору полный обзор местности, с помощью умного подвеса камеры ZT30.

Наше решение решает одну из главных проблем управления роверами, мы можем отправлять команды роверу из любой точки мира, при условии что оба узла подключены к глобальной сети.

Также, что немало важно данное решение очень гибкое, ROS2 дает большие возможности по интеграции других самых разных датчиков, для различных исследований, наш проект это база для будущий проектов.

## Возможности

- полноценное удаленное управление всеми системами по защищенным каналам
- передача изображения (Full HD) с камеры без задержки оператору, путем кодирование и декодирования кадров
- переключение режимов изображения с камеры (Зум, Тепловизор)
- умный управляемый круиз-контроль
- изменение параметров системы (скорость, шаг движения камеры) удаленно
- автомтизированный запуск всех систем на ровере

## Требования

#### Аппаратные требования

- Ноутбук (для оператора)
- Мини-пк (для ровера)
- Камера ZT30
- VESC
- Электромоторы (1 КВт, 2 шт.)
- Pixhawk 2.4.8
- Питание (ровера в целом)
- Модем для сим-карты (для интернет соединения ровера)

#### Системные требования

- Ubuntu 22.04 (и на ровере и на ноутбуке оператора)
- ROS2 Humble
- MavROS
- Ardupilot ArduRover 4.6.3 (прошивка Pixhawk)
- ZeroTier (туннель)

Эта конфигурация нашего ровера, она полностью адаптирована под него, ваша конфигурация может отличаться и требовать дополнительных настроек.

## Установка

В самом начале необходимо клонировать репозиторий 

```bash
git clone https://github.com/fr1cadelka/Aurora_ws.git
```

Далее открываем этот проект в QT Creator. Его необходимо собрать.

Данную процедуру необходимо повторить на обоих ПК.

Далее установите ZeroTier на оба ПК и настройте общую сеть, информациб об этом можно найти в интернете.

#### На ровере

Далее подключите Pixhawk к мини-пк на ровере. Нам необходимо узнать порт на котором он находится.

```bash
ls /dev/ttyACM*
```

Полученный порт необходимо использовать в следующей команде для запуска Pixhawk

```bash
ros2 launch mavros apm.launch fcu_url:=/dev/ttyACM0:115200
```

После успешного запуска Pixhawk, необходимо произвести процедуру ARM

```bash
ros2 service call /mavros/cmd/arming mavros_msgs/srv/CommandBool "{value: true}"
```

Данная команда может не сработать с первого раза, и необходимо повторять ее, пока статус armed не изменится на true. Статус Pixhawk можно проверить следующей командой:

```bash
ros2 topic echo /mavros/state
```

Посел успешного запуска Pixhawk переходим к запуску узлов. Для этого переходим в папку проекта и указываем загрузочный файл.

```bash
cd Aurora_ws/
```

```bash
source install/setup.bash
```

Далее запускаем узел получения изображения с камеры

```bash
ros2 run video_camera_pkg video_camera_pkg_node
```

Если пошли логи, значит все запущено успешно. Далее запускам UDP сервер. server_ip это ip ноутбука оператора, на него будет транслировать видеопоток наш ровер

```bash
ros2 run udp_pkg udp_pkg_server_node --ros-args -p server_ip:=127.0.0.1 -p server_port:=12346
```
